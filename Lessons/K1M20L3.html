Opis klas i obiektów

Podstawową ideą tworzenia klas jest ich powtórne wykorzystanie. Umożliwia to też płynniejsze pisanie kodu przez zespół programistów. Python oferuje zestaw atrybutów, który opisuje klasę:
- __doc__ - opis klasy umieszczony zaraz pod słowem class
- __name__ - nazwa klasy
- __base__ - klasa bazowa (o tym za chwilę)
- __dict__ - słownik zawierający atrybuty i funkcje zdefiniowane w klasie

class Osoba:
'klasa opisująca podstawowe dane osoby'
imie = "imie"
nazwisko = "nazwisko"
wiek = 0
def __init__(self, imie, nazwisko, wiek):
self.imie = imie
self.nazwisko = nazwisko
self.wiek = wiek
def wypisz(self):
print(self.imie,self.nazwisko,self.wiek)
print(Osoba.__doc__)
print(Osoba.__name__)
print(Osoba.__base__)
print(Osoba.__dict__)

Alternatywą do użycia atrybutu __dict__ jest użycie funkcji vars(). W przypadku obiektu zwracany jest słownik ze wszystkimi danymi:

osoba1 = Osoba("Kamil","Bilski",15)
print(vars(osoba1))
{'imie': 'Kamil', 'nazwisko': 'Bilski', 'wiek': 15}

Z kolei funkcja dir() daje listę wszystkich składowych klasy lub obiektu

print(dir(osoba1))
['__class__', '__delattr__', '__dict__', '__dir__', '__doc__', '__eq__', '__format__', '__ge__', '__getattribute__', '__gt__', '__hash__', '__init__', '__init_subclass__', '__le__', '__lt__', '__module__', '__ne__', '__new__', '__reduce__', '__reduce_ex__', '__repr__', '__setattr__', '__sizeof__', '__str__', '__subclasshook__', '__weakref__', 'imie', 'nazwisko', 'wiek', 'wypisz']

Taką listę łatwo odfiltrować z funkcji i atrybutów wbudowanych, z których większość już poznałeś, tak by zostały tylko składowe zdefiniowane przez Ciebie:

skladowe = [s for s in dir(osoba1) if s[0]!='_']
print skladowe
['imie', 'nazwisko', 'wiek', 'wypisz']
